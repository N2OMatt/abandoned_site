Experiência

Game Developer, Fire Horse

Entrei no Fire Horse no final de 2016 como desenvolvedor de jogos. Sendo parte de uma equipe excepcional, tive a oportunidade de trabalhar em projetos incríveis.

Minhas responsabilidades variam entre os projetos que trabalhei, porém um resumo seria:

  • At Sundown (Mild Beast Games): (website)
    • Projetar e implementar intregração do jogo com AlienFX da Dell. O projeto foi escrito em C# com Unity e foi um grande desafio e exercicio de programação e engenharia de software!

  • Killers and Thieves (Alex Thomas / Stoic Studio): (website)
    • Fui responsável por uma grande quantidade de bugs e implementar novas features no projeto. Utilizei Unity e C# para programar e git como versionador de código.

    Este foi um projeto extremamente prazeroso de trabalhar, pois foi ao mesmo tempo muito apertado em relação a prazos, porém de muita apredizagem.
    O projeto contém uma grande quantidade de código legado em Lua, já que ele começou a ser produzido em uma outra engine e a integração com C# foi no minimo, penosa.

    Apesar de não possuir experiência nenhuma em Lua, fui capaz de em pouco tempo fazer as modificações necessárias, e por fim consegui otimizar algumas partes de maneira substancial.

    Por fim, uma feature que me orgulho de ter conseguido implementar com sucesso foi a integração com a Steam. Técnicamente, o desafio não foi muito grande, porém devido ao prazo extreamente curto e a não familiariade com o steamworks eleveram o nivel de dificuldade.

  • Fist Punch 2 (Cartoon Network): (website)
    • Minha responsabilidade era fazer additional programming para melhorar o jogo, por exemplo:
    Adicionar a navegação do teclado aos menus; reestruturar o codigo de localização alguns bug fixes e assim por diante.

    Neste projeto eu utilizei muito os plugins NGUI e GSFU para Unity.

  • Like a Boss (Fire Horse): (website)
    • Fui responsável por uma grande quatidade de pequenas mudanças, refatorações de código, melhorias, correções de bugs e implementação de novos recursos.

    Nesse projeto eu usei os plugins NGUI, DOTween, GSFU.
    Também criei alguns Editor Scripts para facilitar algumas operações de desenvolvimento de jogos.

  • The Saga Banner 1, 2 (Stoic Studio): (website)
    • Em ambos os jogos da Stoic Studio eu fui responsável por manter e melhorar o cógido do projeto bem como o build system escrito por eles.

    A base de código é enorme e é realmente um prazer fazer parte desse projeto. Quando entrei eu não possuia experiência previa com Action Script 3 nem com Mercurial (o SVC usado), elevando assim a complexidade do mesmo.

    Porém fui capaz de aprender as tecnologias e padrões usados de maneira rapida e efetiva, tornando assim este projeto, um dos mais desafiadores e por consequencia mais recompensadores no qual já trabalhei.

  • Let them Come (Klemen Lozar): (website)
    • Neste projeto, fui responsável por integrar o AlienFx SDK ao Game Maker da YoYo Games.
    O AlienFx SDK é escrito em C ++ e fornecido como um arquivo DLL, mas não é prontamente utilizável no Game Maker.

    Meu trabalho foi escrever um "wrapper" o mais 1:1 possivel para mapear as funções C++ em GML (a linguagem de script do Game Maker).

    O projeto foi muito desafiador já que eu não tinha nenhuma experiência em Game Maker, mas no final ocorreu tudo melhor do que o esperado e o projeto foi um sucesso.

  • Outros
    • Na Fire Horse, estamos muito abertos a discussão, de vez em quando eu dou algumas opiniões e idéias sobre o design do jogo, arte, experiência do usuário, etc. Além disso, eu escrevo alguns shell scripts para facilitar o dia a dia do estúdio.

    Ferramentas

    • Amazing Build
      • Conjunto de ferramentas para agilizar o Content Pipeline em Jogos.
      Utiliza o ExtendScript Toolkit (Javascript) da Adobe para manipular os arquivos do Photoshop, Python e Shell Script para outras tarefas.
      Utiliza Python e ShellScript para manipular os arquivos do GIMP.
    • TSP
      • Texture Sheet Packer (Empacotador de texturas), feito em QT e C++. Atualmente possui apenas um exporter de plist para o Cocos2d-x, porém o projeto é arquitetado de maneira a suportar trivialmente a adição de outros exporters e algoritmos de bin packing.
    • Bin2Header
      • Ferramenta para criar um C/C++ header files com o conteúdo de qualquer arquivo dado em uma array de valores hexadecimais.
      Escrito em C.
    • cowss
      • Programa para capturar a tela do desktop (Screenshot) extremamente simples.
      Criado utilizando X11 e CImg.
    • COWTODO
      • Extrai do código-fonte, todas as cow tags e as imprimi de uma forma muito organizada.
      Escrito em Python.
    • Dummy Textures
    • FrameMerger
      • Ferramenta para mesclar (merge) uma imagem em várias outras images localizada no diretório especificado.
      Escrito em Python, utilizando Pygame e PyQt4.
    • Gosh
      • Programa que faz com que a navegação shell um pouco mais fácil, permitindo-lhe atribuir nomes significativos aos caminhos (paths) e navegar usando esses nomes significativos em vez dos caminhos brutos.
      Escrito em Python e Shell Script.
    • GuardChecker
      • Verifica e aplica consistência em include guards de C e C++.
      Escrito em Python.
    • ImageConveter
      • Ferramenta para converter formato de imagens.
      Escrito em Python, utilizando Pygame.
    • ImageOrganizer
      • Ferramenta para organizar imagens em diretórios estruturados, classificadas por tamanhos e/ou formato das imagens.
      As imagens (de entrada e saída) deve estar em qualquer formato de imagem suportado pelo PyGame.
      Escrito em Python, utilizando Pygame.
    • Linux_USBBootCreator
      • Ferramenta para ajudar a tarefa fácil, mas muito tediosa, de transformar um pendrive em um pendrive bootavel.
      É basicamente um wrapper em torno do dd(1) permitindo que você esqueça sobre as desajeitadas opções de linha de comando e fornecendo verificações de segurança.
      Escrito em Python.
    • OSX_USBBootCreator
      • A mesma funcionalidade que Linux_USBBootCreator porém para OSX.
      Escrito em Python.
    • super-rename
      • Programa que altera o conteúdo e/ou renomeio os arquivos que correspondem a um determinado regex.
      Escrito em Shell Script.
